# Setup @mmaappss/sync for local testing

Use this skill when the developer wants to **link or pack** @mmaappss/sync into another project for local testing. Run the full flow: wizard, wire-up, **link (or pack) commands**, and **verify** — do not hand off to the user to run link or sync unless tool access to the target path is blocked.

## When to use

- User says "test locally", "link mmaappss", "pack and test", or "set up sync in another project"
- User is developing mmaappss and needs to verify it works in a consumer repo

## Wizard flow / AskUserQuestion / Write imperative steps with explicit inputs and outputs.

Collect these **inputs** in order using **AskQuestion**. For choice fields, present **only** the allowed values (radio-style); do not accept open-ended input for those fields. Resolve the target path to an absolute path and use it for all later steps.

| 2 | **Install method** | "Install @mmaappss/sync via?" | radio | **Only** `pnpm link` (quick) **or** `npm pack` then `pnpm add file:...` (publish-like). Use AskQuestion with exactly these two options. |

## Wire-up steps

**Sample plugin:** Copy `packages/sync/getting-started/plugins/git` into `<target>/.agents/plugins/git`. If add-to-existing, copy only if `git` plugin is not present.

**Config:**
- Env: Create or append to `<target>/.env` from `packages/sync/getting-started/env.example`. For add-to-existing, append only missing MMAAPPSS_* lines.
- TypeScript: If greenfield, write `<target>/mmaappss.config.ts` from `packages/sync/getting-started/mmaappss.config.example.ts`. If add-to-existing, merge marketplacesEnabled (ensure claude/cursor/codex enabled).

**Script:** Add to target package.json: `"mmaappss:sync": "mmaappss-sync"`.

**Install (run these, do not only suggest):**
- If link: (1) From mmaappss repo `packages/sync`, run `pnpm link --global`. (2) From target root, run `pnpm link @mmaappss/sync`.
- If pack: Run `npm pack` in `packages/sync`, then in target `pnpm add file:.../mmaappss-sync-0.1.0-beta.1.tgz` (use actual tgz path).

## Verify (run and report)

From **target root**, run `pnpm run mmaappss:sync` (or `pnpm exec mmaappss-sync`). Then check and report:
- **Claude:** `<target>/.claude-plugin/marketplace.json` exists and lists plugins → "Claude: OK" or "Claude: missing"
- **Cursor:** `<target>/.cursor/rules/` or manifest `.cursor/.mmaappss-cursor-sync.json` has content → "Cursor: OK" or "Cursor: missing"
- **Codex:** `<target>/AGENTS.override.md` has section `## packages (generated by mmaappss)` → "Codex: OK" or "Codex: missing"

If tool access to the target is blocked, output the verify command and the three checks for the user to run.

## Linked install (no hacks needed)

The bin (`mmaappss-sync`) is built to work when the package is **pnpm linked**: it uses this package's tsx loader (absolute path) so the target does not need tsx installed, and it sets `MMAAPPSS_REPO_ROOT` to the current working directory so repo root is the target project, not the mmaappss repo. Do not add workarounds; the code already handles linked installs.

## Edge cases

- Target has no package.json: skill can say "target should be a pnpm project"; link/add may create one or fail
- Config merge for add-to-existing: append env; for TS, ensure marketplacesEnabled includes claude/cursor/codex
- Agent cannot read/write target: complete wizard, then output exact commands and file contents (or paths to copy) for the user
