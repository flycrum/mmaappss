# Remove @mmaappss/sync

Use this skill when the developer wants to **uninstall** mmaappss sync and remove configs/artifacts.

## When to use

- User says "remove mmaappss", "uninstall sync", "teardown mmaappss", or "get rid of mmaappss"
- User wants to switch away from mmaappss sync

## Wizard flow

1. **Target project** — Use cwd or prompt for path.
2. **Install type** — AskQuestion or infer: "Was @mmaappss/sync installed via?" → `pnpm link` | `npm/pnpm add` (regular install).
3. **Scope** — AskQuestion: "Remove?" → `Full teardown` (config, package.json dep, outputs) | `Config only` (keep package, remove config and sync outputs) | `Outputs only` (clear .claude-plugin, .cursor, AGENTS.override.md section but keep config).

## Teardown steps (full)

1. **Remove from package.json** — Remove `@mmaappss/sync` from dependencies. Remove mmaappss:sync script. Run `pnpm install` to update lockfile.
2. **Remove config** — Delete mmaappss.config.ts; remove MMAAPPSS_* lines from .env (or delete .env if it only had those).
3. **Clear outputs** — Remove or clear:
   - `.claude-plugin/marketplace.json` (or whole .claude-plugin if only mmaappss)
   - `.cursor/rules/`, `.cursor/commands/`, `.cursor/skills/`, `.cursor/agents/` (mmaappss-synced content); `.cursor/.mmaappss-cursor-sync.json`
   - AGENTS.override.md section `## packages (generated by mmaappss)` (or whole file if only that)
4. **Optional: remove .agents/plugins** — If user wants to remove sample plugins (e.g. git from getting-started), offer to delete them.

**If link:** Run `pnpm unlink @mmaappss/sync` from target before removing from package.json.

## Notes

- Do not remove user-authored rules/commands/skills; only mmaappss-synced content. When in doubt, ask.
- Config only: remove mmaappss.config.ts and MMAAPPSS_* from .env; keep package and script for later re-config.
- Outputs only: run `mmaappss-sync:clear:all`-style logic or manually remove sync artifacts.
