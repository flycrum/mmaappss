# getting-started plugin

Setup, diagnose, update, and remove **@mmaappss/sync** in consumer projects.

- **setup-local-testing:** Use when the user wants to test mmaappss sync locally via link or pack. [getting-started-setup-local-testing](skills/getting-started-setup-local-testing/SKILL.md)
- **setup-from-npm:** Use after `pnpm add @mmaappss/sync`. [getting-started-setup-from-npm](skills/getting-started-setup-from-npm/SKILL.md)
- **doctor:** Use when sync fails or outputs are missing; investigate config, versions, schemas. [getting-started-doctor](skills/getting-started-doctor/SKILL.md)
- **updates:** Use when refreshing sample plugins or templates in the target. [getting-started-updates](skills/getting-started-updates/SKILL.md)
- **remove-mmaappss-sync:** Use when uninstalling mmaappss sync and removing configs/artifacts. [getting-started-remove-mmaappss-sync](skills/getting-started-remove-mmaappss-sync/SKILL.md)
